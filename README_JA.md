# Work Time Tracker — リモートチーム向け作業時間管理ツール

> リモートチームのために設計されたデスクトップアプリ。Notionをタスク管理と時間記録の中枢として活用し、作業時間とスクリーンショットを自動同期することで、リモートワークの透明性を高めます。

**Language / 语言 / 言語:** [English](./README.md) | [中文](./README_CN.md) | [日本語](./README_JA.md)

---

## どんな課題を解決するのか

リモートワーク最大の課題のひとつは**工数の不透明さ**です。タスクは口頭で割り当てられ、作業時間の把握が難しく、請求根拠が曖昧になりがちです。

Work Time Tracker は Notion と連携し、各メンバーの作業時間とスクリーンショットを自動的に Notion データベースへ同期します。プロジェクトマネージャーはリアルタイムで各メンバーの工数を把握でき、タスク配分・工数管理・請求計算を正確に行えます。

## 主な機能

### タスク管理
- Notion データベースからタスク一覧を自動取得（「未着手」「進行中」を対象）
- 担当者・締切日・累計作業時間を表示
- ワンクリックで最新状態に更新

### 時間トラッキング
- タスクを選択してワンクリックで計測開始／停止
- 複数セッションの時間を累積集計し Notion に自動反映
- **時間データを最優先で同期** — 請求計算の精度を保証

### 自動スクリーンショット
- 作業中、5分ごとに自動でスクリーンショットを撮影
- セッション内の全スクリーンショットを2列のコラージュ画像に合成して Notion へアップロード（画像数の爆発を防止）
- Notion 上では直近20枚のコラージュを保持し、古いものは自動削除

### 安定したオフライン同期
- **ローカル優先**：計測停止直後にデータを `pending_uploads.json` へ書き込み — ネットワーク障害やクラッシュでもデータは失われない
- **時間とスクリーンショットを分離**：スクリーンショットのアップロードが失敗しても、時間データは確実に同期
- **無限リトライ**：バックグラウンドスレッドが2分ごとに自動リトライ — ネットワーク復旧後は手動操作不要
- **起動時の自動再開**：前回セッションの未完了アップロードを起動時に自動検出して再試行
- **リアルタイムフィードバック**：「時間同期済み ✓」「スクリーンショットをバックグラウンドで再試行中」など状況を随時表示

## 設定方法

アプリは以下の順で `config.yaml` を探します：

| 優先度 | パス | 推奨対象 |
|--------|------|----------|
| 1番目 | `~/.work_time/config.yaml` | 全ユーザー（exe・ソース共通） |
| 2番目 | `./config.yaml` | 開発・デバッグ時のみ |

### ステップ1 — 設定ディレクトリとファイルを作成

**Windows：**
```
mkdir %USERPROFILE%\.work_time
copy config.yaml.example %USERPROFILE%\.work_time\config.yaml
```
`C:\Users\<ユーザー名>\.work_time\config.yaml` をメモ帳等で編集してください。

**macOS / Linux：**
```bash
mkdir -p ~/.work_time
cp config.yaml.example ~/.work_time/config.yaml
```

### ステップ2 — Notion の認証情報を入力

```yaml
notion:
  api: "secret_xxxxxxxxxxxxxxxxxxxx"          # Notion Integration Token
  database_id: "xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"  # データベース ID（32文字）

# Notion データベースのフィールド名と完全一致させること
database:
  task_name: タスク名
  assignee: 担当者
  status: ステータス
  time_spent: 作業時間（分）
  screenshots: スクリーンショット
  due_date: 期限
```

### ステップ3 — Notion の認証情報を取得

**Integration Token：**
1. https://www.notion.so/my-integrations にアクセス
2. **「新しいインテグレーション」** をクリックして作成
3. **インターナルインテグレーションシークレット**（`secret_` で始まる文字列）をコピー

**データベース ID：**
1. ブラウザで Notion のデータベースページを開く
2. URL の形式：`https://www.notion.so/xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx?v=...`
3. `.so/` と `?v=` の間にある32文字の文字列がデータベース ID

**インテグレーションをデータベースに接続：**
1. Notion データベースページ右上の `...` → **接続** → 作成したインテグレーションを選択

## Notion データベースの設定

以下のフィールドを持つデータベースを Notion で作成してください：

| フィールド名 | タイプ | 説明 |
|------------|--------|------|
| タスク名 | Title | タスクのタイトル |
| 担当者 | Select | 担当メンバー |
| ステータス | Status | 未着手 / 進行中 / 完了 |
| 作業時間（分） | Number | 累計作業時間（分）— 自動更新 |
| スクリーンショット | Files | セッションコラージュ — 自動アップロード |
| 期限 | Date | タスクの締切日 |

> フィールド名は `config.yaml` の `database` セクションで自由にマッピング可能です。

## 動作フロー

```
Notion データベース（タスク割り当て）
        ↓  タスク一覧を取得
Work Time Tracker（デスクトップ）
        ↓  計測開始 + 自動スクリーンショット
        ↓  停止後すぐにローカル保存
        ↓  バックグラウンド同期（無限リトライ）
Notion データベース（工数 + スクリーンショット更新）
        ↓
マネージャーが各メンバーの工数をリアルタイム確認
```

## ローカルデータファイル

| ファイル | 説明 |
|---------|------|
| `~/.work_time/config.yaml` | Notion 認証情報とフィールドマッピング設定 |
| `pending_uploads.json` | 同期待ちキュー — 未アップロードセッションを記録 |
| `screenshots/` | スクリーンショット保存ディレクトリ |
| `work_tracker.log` | 同期ログ（エラー・リトライ状況を含む） |
